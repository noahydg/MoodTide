# MoodTide 心潮 · 完整方案

> 抖音 AI 创变者计划黑客松联赛 · 温州站 · **赛道三「AI 体验：刷到懂你的瞬间」**
> 载体：移动端 Web H5（抖音视觉风格）

---

## 1. 核心重构（拿奖关键）

赛道三要的不是“打开后生成音乐的 App”，而是 **一张刷到就成立的信息流卡片**。
MoodTide 定位重写为：

> **一张「会呼吸的情绪卡片」**：在你刷到它的 0.5 秒里，它已读出屏幕外的此刻
> （时段 / 天气 / 节气），并当场为这一刻、只为你，拉起一段不会重复的旋律。
> 你什么都不用做，音乐就已经在响；你唯一的一次互动，是 **按住屏幕把这段心潮「收下」**，
> 松手则任它退潮消失。

把“情绪→音乐”从一个**功能**，压成一张 **「刷到就在响、按一下就带走、刷走即焚」** 的活体卡片。
这一句同时命中赛道三全部要点：零跳转、零输入、刷到即懂、一次轻互动、用完即走、持续变化。

### 两层结构（团队确认的产品骨架）

> 不是“纯刷走即焚、不点进去”，而是 **两层**：

- **第一层 · 信息流卡片** ＝ 被刷到那一刻 **怎样吸引你进来**：刷到即响、读心文案、声波海面、
  长按可直接收下。这是赛道三的“刷到即成立”，**永远零延迟，绝不 loading**。
- **第二层 · 点进去的详情** ＝ 进来之后 **怎样留住你、给更深体验**：情绪轮微调旋律、
  “✨为我真实生成专属曲”（豆包音乐，异步，有兜底）、收下到潮汐瓶、生成分享卡。
  这是“功能点都 cover”的承载层，把传播/技术/商业的料都放这里。

## 2. 交互闭环（刷到→理解→打动→一次互动→用完即走）

1. **刷到（0–0.5s）**：卡片进入视口即自动起调，无需点击。卡面是随声起伏的“声波海面”，
   顶部一行轻量情境签名「5月30日 · 小满 · 黄昏」，色调随当下情境实时上色。
2. **理解（0.5–2s）**：浮现一句不超过 12 字的“读心文案”（如“像加完班松了口气的那种安静”）。
   它不问你心情，而是替你说出心情——“你不必开口”被一眼证明。
3. **被打动（2–8s）**：旋律进入主句，声波海面与音乐严丝合缝起伏（**视觉由音频实时驱动**）。
   角落小字“第 N 段 · 此刻独有 · 潮起潮落从不重复”，钉死稀缺感。
4. **一次互动（唯一）**：长按屏幕 = 把这段心潮“收下”，声波被吸成一颗潮汐贝壳进入“我的潮汐瓶”；
   松手不按则旋律自然退潮、归于静默海面。只有一个手势，没有第二个决策。
5. **用完即走**：松手或上滑即走，无弹窗、无注册、无“是否保存”追问。下次再刷到时卡片会
   呼应“上次你在雨夜收下过一段，今天的潮不一样了”——靠**持续变化 + 记得你**制造“下次再遇见”。

## 3. 高光时刻（演示夺冠点）

评委拇指停在卡片上、**还没点任何按钮**——一段贴合“此刻”的旋律已淡入：卡顶显示真实当下
情境，配色随之上色，声波海面随旋律实时起伏。“它怎么知道现在是这个氛围？”——这就是被理解的瞬间。
紧接着浮出“这一段，只在此刻为你而生，刷走就再也听不到”，制造唯一性 + 稍纵即逝的心动。

## 4. 关键技术决策

| 维度 | 决策 | 理由 |
|---|---|---|
| 载体 | 移动端 Web H5（抖音暗色风格、竖屏全屏 scroll-snap） | 48h 最稳、本机一键预览、扫码即体验；赛道三不强制形态（赛道二才要小程序码） |
| 框架 | Vite + TypeScript，**无框架** | 自定义音频可视化最自由、零开销 |
| 可视化 | **Web Audio `AnalyserNode` 实时驱动 Canvas 声波海面** | “不是贴的循环动画”的技术铁证，评委一眼可辨 |
| 情绪捕捉（主） | **环境上下文自动感知**（时间/节气/可选天气）→ 情绪档位 | 零打字、最稳、最契合“刷到即懂” |
| 情绪捕捉（辅） | **长按收下** 这一个手势 | 唯一互动，不加第二个决策 |
| 音乐·第一层（信息流） | **合成器/预生成 mp3 池**，零延迟 | 刷到即响，现场**绝不能有 loading**，延迟即死亡 |
| 音乐·第二层（详情·主路径） | **豆包音乐生成模型**（火山 Seed-Music/豆包·音乐）真实当场生成 | 团队定为主路径；境内、无需代理；异步，藏在“主动点生成”里 |
| 音乐·第二层（备路径） | **sunoapi.org**（Suno 第三方代理） | 音质最高的备选；需海外网络+充值 |
| 音乐兜底（永不挂） | mp3 池 → 合成器 | 任一在线引擎失败/超时无缝下沉，保证必出声 |
| 音源版权 | **Suno 生成**，分享物注明“AI 生成”+ MoodTide 水印 | 团队确认；规避抖音平台版权检测 |
| 文案（加分） | **豆包大模型 API（火山方舟 ARK）** 生成读心文案/作曲 prompt | 中国境内最稳，OpenAI 兼容；key 走 BFF 不进前端 |

> 四级降级链（`src/music/musicService.ts` 已实现）：
> **豆包音乐（主）→ Suno 代理（备）→ mp3 池 → 合成器（永不挂）**。第一层只走后两级零延迟来源。

### 音乐 API 现状（调研结论）
- **Suno 官方开发者通道**：门槛高（海外账号+支付+可能 waitlist），48h 内不稳，**不作主路径**。
- **海绵音乐**：是 C 端 App，**无公开开发者 API**；字节 Seed-Music 走火山引擎 B 端需申请。
- **sunoapi.org（第三方代理）**：注册充值即拿 key，异步（提交 taskId → 轮询，~30s 出首段流式），
  仅适合黑客松 Demo，**国内访问需后端代理、前端不要直连**。
- **豆包 ARK**：境内最稳，秒级，新用户有免费额度，用于文案/prompt 几乎零成本。

## 5. 评分维度对策

- **产品完成度（看“能否被评委亲手刷到并跑通”）**：减法到只剩一张卡片的闭环（自动起调→读心→
  声波实时起伏→长按收下→退潮），0 bug、60fps。不做登录/设置/曲库页。
- **技术实现**：押两处肉眼可见的真技术——① Analyser 实时驱动声波；② 情境引擎现场改时间/隔天
  刷会真的变。架构图把 input 从“用户指令”改为“环境信号”，正是“你不必开口”的技术表达。
- **创新性**：一句话“**第一个把 AI 音乐从生成工具变成刷到即焚的信息流卡片**”。三反直觉：
  零输入（读你而非问你）、稍纵即逝（刷走即焚、每段唯一）、视觉由声音长出。明确讲“砍掉了什么”。
- **传播潜力**：每段卡片一键录屏成竖屏短视频，带情境水印“此段全网仅你一份”；话题
  “#刷到只属于此刻的歌”；强调“下次再遇见会不一样”制造复访。
- **商业价值（各一句，不展开）**：① 情境化品牌/文旅联名卡片（原生信息流广告）；
  ② “潮汐瓶”会员永久保存稍纵即逝的旋律（FOMO 变订阅）；③ 远期 link→cure 音乐疗愈（仅点到）。

## 6. 演示视频脚本（≤4 分钟，90% 给“刷到即响”的真实体验）

- **0:00–0:20 钉定位**：黑屏出 Slogan，旁白“市面 AI 音乐都要你先开口、先等待、先点进去——
  我们做的是刷到那一刻就在响的一张卡片”。
- **0:20–1:10 高光·刷到即懂**：上滑刷到卡片，旋律立刻淡入，卡顶真实当下情境，声波随音律起伏，
  浮出读心文案。旁白一句“我没点任何东西，它已经知道现在是什么感觉”。留静默让音乐说话。
- **1:10–1:50 一次互动·收下与退潮**：长按收下→贝壳飞入潮汐瓶；下一张旁白“不收下就退潮，永远消失”。
- **1:50–2:40 技术可信·持续变化是真的**：现场改系统时间/切雨天，重刷卡片，文案/配色/旋律全变；
  快闪架构图：环境信号→情境引擎→实时混音→声波可视化。
- **2:40–3:20 传播与连续**：一键录屏成带水印短视频；第二次刷到“卡片记得你上次收下过”。
- **3:20–3:50 商业与收口**：三张联名卡片快闪 + 一句话商业，落版 Slogan“潮起潮落，从不重复”。

## 8. 风险与红线（敢做减法）

- **延迟即死亡**：刷到后任何可见 loading = “刷到即成立”当场破功。生成永远藏在体验之外。
- **可视化不能造假**：声波必须 Analyser 真实驱动，演示可现场切歌让评委看到它随之变。
- **贪多稀释主线**：cure lab / 医院疗愈 / 产品化在 Demo 只各留一句话，绝不展开。
- **音乐质量天花板**：升级真 mp3 时，宁可每档 1–2 条极好听，也不要一堆平庸雷同。
- **读心翻车**：情境→情绪只做几个高置信度档位（深夜安静/黄昏松弛/雨天微忧…），**准比全重要**。
- **稍纵即逝 vs 想再听**：把“想再听却没了”的懊恼，转化为“长按收下”与“潮汐瓶付费”的动力——讲成卖点而非 bug。
- **合规**：UGC/AI 内容注意敏感词与版权；真生成音频标注 AI 生成；演示数据自备、勿涉真人隐私（视觉情绪识别为可选项，慎用）。

## 9. 待团队拍板

1. 加分项优先级：**豆包文案** vs **预生成 mp3 池** vs **Suno 真生成**——建议按此序，时间不够就只做第一个。
2. 是否启用天气信号（需一个免费天气 API + 定位授权）——MVP 可先不接，仅用时间+节气。
3. 是否要 BFF（接豆包/Suno 才需要）——纯合成器 Demo 不需要后端。
4. 视觉情绪识别（自拍/表情）是否做——隐私与稳定性风险，列为最低优先、慎用。
