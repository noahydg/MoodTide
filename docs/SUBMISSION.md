# MoodTide 提交流程（抖音虚拟创作平台 · 互动空间）

> 提交载体已定：**抖音虚拟创作平台 → 作品管理 → 互动空间 → 创建作品**。
> 上传 **ZIP 包**（或粘贴 HTML 代码），**竖屏体验**。本文是从开发到发布的总体流程。

---

## 0. 平台硬约束（决定我们怎么构建）

| 约束 | 数值/要求 | 我们的对策 | 状态 |
|---|---|---|---|
| 入口文件 | ZIP **根目录**必须有 `index.html` | `pack.sh` 在 `dist/` 内打包，保证 index.html 在 zip 根 | ✅ |
| 包体大小 | **≤ 8MB** | 合成器零音频素材，打包 ~30KB；mp3 走压缩或 OSS | ✅ 远低于上限 |
| 资源路径 | 沙箱任意目录运行 | `vite base: './'`（相对路径）+ 小资源内联 | ✅ |
| 体验方向 | **竖屏** | 全程移动端竖屏设计（100dvh + scroll-snap） | ✅ |
| 外部网络 | **不确定**沙箱是否放行 fetch/HTTPS | 见下「网络策略」——提交版默认离线自包含 | ⚠️ 待现场验证 |
| 必填元数据 | 标题≤20字、描述≤200字、产物来源、作品图标 | 见下「元数据」 | 📝 待定稿 |

### 网络策略（关键决策）

平台沙箱**是否允许外部网络请求未知**，因此分两套：

- **提交版（保命）**：100% 自包含、离线可跑。音乐 = Web Audio 合成器（零素材），
  情绪推断 = 纯前端（时间/节气/迷你信息流）。**无任何外部请求**，沙箱必跑通。
- **增强版（加分）**：若沙箱放行 HTTPS，则 `musicService` 在第二层调豆包音乐真生成
  （走 BFF）。失败/被拦自动降级回合成器（四级降级链已实现），用户无感。
- **演示视频**：无论沙箱如何，豆包真生成都在录屏里完整展示，确保技术分拿到。

> ⏱️ 现场第一件事：上传一个最小测试包，验证沙箱能否 `fetch` 外网，再决定增强版是否上。

## 1. 开发 → 打包 → 上传 全流程

```
┌─ 开发 ──────────────────────────────────────────────┐
│ npm run dev            本机预览                        │
│ npm run dev -- --host  手机真机(同局域网)             │
└──────────────────────────────────────────────────────┘
        │
        ▼
┌─ 打包 ──────────────────────────────────────────────┐
│ npm run pack                                          │
│   = npm run build (tsc + vite，相对路径)             │
│   + scripts/pack.sh                                   │
│       · 在 dist/ 内 zip，index.html 落在 zip 根       │
│       · 校验根目录 index.html 存在                    │
│       · 校验 ≤ 8MB，超限报警                          │
│   产出：moodtide-submit.zip                           │
└──────────────────────────────────────────────────────┘
        │
        ▼
┌─ 上传（平台界面，按官方图示）─────────────────────────┐
│ 1. 进入「互动空间」管理列表                           │
│ 2. 右上角「创建作品」                                 │
│ 3. 填标题/描述/产物来源 → 选「竖屏体验」              │
│    → 上传作品图标 → 上传 moodtide-submit.zip          │
│    （上传卡住可取消重传）                             │
│ 4. 点「创建」                                         │
│ 5. 「扫码测试」用手机抖音扫码真机验收                 │
│ 6. 确认无误 →「发布」                                 │
└──────────────────────────────────────────────────────┘
```

备选：若 zip 上传不顺，可用 **单文件 HTML 粘贴**（后续可加 `vite-plugin-singlefile`
把整站内联成一个 index.html 直接粘贴；当前 JS 体积极小，完全可行）。

## 2. 元数据（待团队定稿，先给草案）

- **作品标题**（≤20字）：`MoodTide 心潮 · 刷到懂你的潮`（草案）
- **描述**（≤200字）：`你不必开口。MoodTide 从此刻的天气、时间、你刚刷过的内容，
  读出你当下的心境，当场升起一段只属于这一刻、不会重复的旋律——刷到就在响，
  按一下就带走，刷走即焚。心是月，乐是潮。`（草案，约 80 字）
- **产物来源**：按平台选项填（如 Trae / Claude Code 等 AI 辅助）。
- **作品图标**：竖屏潮汐主题，暗底 + 月与声波（可用豆包/即梦生成）。

## 3. 提交前自检清单

- [ ] `npm run pack` 通过，moodtide-submit.zip 根目录有 index.html
- [ ] 包体 ≤ 8MB
- [ ] 断网状态下本地 `npm run preview` 能完整跑通（验证自包含）
- [ ] 竖屏布局在真机无溢出、安全区（刘海/底部）正常
- [ ] 首次进入「轻触解锁音频」后能出声（autoplay 策略）
- [ ] 迷你信息流→心潮卡→详情 全链路无报错、无 loading 转圈
- [ ] 扫码测试通过后再发布
